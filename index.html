Philosophy

  • Build big things out of predictable, simple, small things.
  • Isolate visual component functionality into re-usable HTML elements.
  • Extend the Web platform rather than build to a framework.
  • No special compiler or language extensions.
  • Scale down to a script include, scale up as an npm module.
  • Use functional programming patterns for render pipelines.

GZA: Functional HTML Elements

```js const gza = require('gza') gza` ${(settings, innerHTML) ⇒ `${innerHTML}`} ` ```

Simple functional API for defining and rendering custom HTML elements.

Dyanmic Rendering

Setting intialization and changes.

You can define settings, along with default values, for your element.

Any attribute or property changes on your element to those values will trigger a re-render of the element.

```javascript const gza = require('gza') gza` ＜wrap-counter ${{counter: 0}}＞ ${settings ⇒ `Counter: ${settings.counter}`} ＜/wrap-counter＞ ` document.body.innerHTML += '＜wrap-counter＞＜/wrap-counter＞' let el = document.querySelector('wrap-counter') setInterval(() ⇒ el.counter += 1, 1000) ```

Defining Settings

  • Set a default using
    gza`<my-element ${{prop: 'default'}}>`
  • Activate a list of settings to be set later by element attributes/properties.
    gza`<my-element ${['propname']}>`
  • Call
    element.addSetting(key, value)
  • Create a promise for an eventual setting
    element.waitFor(key)

Changing Settings

  • Set element attributes that have been previously defined as settings.
    • <my-element prop="pass">
    • element.setAttribute('prop', 'pass')
  • Set element attributes that have been previously defined as settings.
    • element.prop = 'pass'

Requiring State

There may be some state that you absolutely require before working.

Since rza accepts async functions and thenabales in all initialization and template functions you can effectively block rendering at any stage by waiting on a promise.

Notice that none of the content in the template is rendered until the promise is resolved.

```javascript const gza = require('gza') gza` ＜manual-count＞ ＜span＞Counter: ＜/span＞ ${settings ⇒ settings.waitFor('counter')} ＜span＞ | ＜/span＞ ＜/manual-count＞ ` document.body.innerHTML += '＜manual-count＞＜/＜manual-count＞' let i = 0 let el = document.querySelector('manual-count') let button = document.createElement('button') button.onclick = () ⇒ el.counter = i = i++ document.body.appendChild(button) ```
